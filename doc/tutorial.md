# MNIST CNN 架构设计原理

## 为什么参数少反而好？

旧架构 315K 参数，新架构 49K（缩小 6 倍），但准确率从 97.6% 升到 98.4%，手写泛化也好得多。

核心原因：**旧模型把大部分参数塞在全连接层里背训练集，新模型用更多卷积层学形状。**

### 旧架构的问题

```
Input (1×28×28)
    ↓
Conv2D(1→32, k=3, p=1)     → 32×28×28    320 参数
    ↓ ReLU
MaxPool2D(2×2)              → 32×14×14
    ↓
Conv2D(32→64, k=3, p=0)    → 64×12×12    18,496 参数
    ↓ ReLU
MaxPool2D(2×2)              → 64×6×6
    ↓
Flatten                      → 2304
    ↓
Linear(2304→128)             → 128         294,912 参数
    ↓ ReLU
Linear(128→10)               → 10          1,290 参数
──────────────────────────────────────────
总参数: ~315,000
```

旧架构只有 2 层卷积（32 和 64 个滤波器），但每个滤波器较大。经过两次 MaxPool 后，Flatten 层展开得到 2304 维特征向量。关键瓶颈：全连接层 FC(2304→128) 有 **295K 参数**，占总参数 94%。

这 295K 个权重中，每个权重都绑定到一个**特定的空间位置**——"第 3 行第 7 列的通道 5"和"第 3 行第 8 列的同一个通道"是独立的两个权重。模型实际上是在记忆"数字在哪个位置长什么样"，而不是"数字本身长什么样"。

此外，第一层卷积使用了 padding=1 来保持空间尺寸，这意味着边缘像素被零填充包围，产生了边界伪影。没有数据增强、Dropout、Momentum——模型 5 个 epoch 就能达到 97.6%，但这只是因为它记住了 MNIST 中数字的标准位置和大小。一旦手写数字略有偏移、旋转或缩放，这些位置相关的权重全部失效。

### 新架构的设计

```
Conv2D(1→10) → ReLU → Conv2D(10→10) → ReLU → MaxPool →
Conv2D(10→20) → ReLU → Conv2D(20→20) → ReLU → MaxPool →
Flatten(320) → Dropout(0.2) → FC(128) → ReLU → FC(10)
```

**4 个卷积层堆叠**（无 padding），每层滤波器很少（10→10→20→20）。整个卷积部分只有 ~6.5K 参数。

关键设计：

1. **深度优先**：4 层 3×3 卷积堆叠，顶层的每个神经元能看到输入图像中 **16×16** 的感受野（旧架构只有 10×10）。更大的感受野 = 更多的上下文 = 更好的形状识别。

2. **空间压缩**：每层卷积让特征图缩小 2 像素（28→26→24→Pool→12→10→8→Pool→4）。到 Flatten 时只有 320 维（旧架构是 2304 维），全连接层的参数量降到 41K。

3. **卷积做重活**：4 层卷积逐层抽象——第 1 层检测边缘，第 2 层组合成曲线，第 3 层识别局部形状，第 4 层整合成完整数字。全连接层只负责最后的分类投票。

## 架构详情

### 层规格

| 层 | 类型 | 输入 | 输出 | 参数量 | 感受野 |
|-----|------|------|------|--------|--------|
| 1 | Conv2D 3×3 | 1×28×28 | 10×26×26 | 100 | 3×3 |
| 2 | ReLU | — | 10×26×26 | 0 | — |
| 3 | Conv2D 3×3 | 10×26×26 | 10×24×24 | 910 | 5×5 |
| 4 | ReLU | — | 10×24×24 | 0 | — |
| 5 | MaxPool2D 2×2 | 10×24×24 | 10×12×12 | 0 | 6×6 |
| 6 | Conv2D 3×3 | 10×12×12 | 20×10×10 | 1,820 | 10×10 |
| 7 | ReLU | — | 20×10×10 | 0 | — |
| 8 | Conv2D 3×3 | 20×10×10 | 20×8×8 | 3,620 | 16×16 |
| 9 | ReLU | — | 20×8×8 | 0 | — |
| 10 | MaxPool2D 2×2 | 20×8×8 | 20×4×4 | 0 | 18×18 |
| 11 | Flatten | 20×4×4 | 320 | 0 | — |
| 12 | Dropout p=0.2 | 320 | 320 | 0 | — |
| 13 | Linear | 320 | 128 | 41,088 | — |
| 14 | ReLU | 128 | 128 | 0 | — |
| 15 | Linear | 128 | 10 | 1,290 | — |

**总参数量：48,828**

### 参数分布

```
卷积部分:   6,450 (13.2%)
全连接部分: 42,378 (86.8%)
─────────────────────
总计:      48,828
```

旧架构全连接占 94%，新架构占 87%。虽然比例接近，但绝对数量差距巨大：新全连接层只有 42K 参数 (vs 296K)，因为它接收的输入维度从 2304 降到了 320。

## 训练配置

| 参数 | 值 | 说明 |
|------|-----|------|
| 学习率 | 0.001 | 初始学习率，配合 momentum 使用 |
| Momentum | 0.9 | SGD 动量，加速收敛 |
| 衰减率 | 0.95/epoch | 每轮衰减，避免末期震荡 |
| Batch size | 64 | 小批量 SGD |
| Dropout | 0.2 | 仅在 Flatten 后、FC 前 |
| 数据增强 | 旋转±15°、缩放±15%、平移±2.8px | 关键：让模型学会形状而非位置 |

### CrossEntropyLoss

模型最后一层输出 10 维 logits（不经过 softmax）。CrossEntropyLoss 内部计算 softmax + NLL：

```
loss = -log(softmax(logits)[label])
grad = softmax(logits) - one_hot(label)
```

### 数据增强

训练时每张图随机做：

- **旋转**：以图像中心旋转 ±15°，双线性插值
- **缩放**：0.85x ~ 1.15x，双线性插值
- **平移**：x 和 y 方向各 ±2.8 像素

这些变换模拟了真实手写中不可避免的位置、角度和大小差异。没有数据增强时，模型在标准 MNIST 测试集上也能达到 98.8%，但对手写数字几乎不识别——因为它只见过完美居中的数字。

## 训练结果

| Epoch | Train Loss | Train Acc | Test Acc |
|-------|-----------|-----------|----------|
| 1 | 0.5448 | 82.6% | 96.8% |
| 2 | 0.2366 | 92.9% | 97.7% |
| 3 | 0.1874 | 94.4% | 97.9% |
| 4 | 0.1678 | 94.9% | 98.3% |
| 5 | 0.1466 | 95.6% | 98.4% |

注意训练准确率明显低于测试准确率（95.6% vs 98.4%）——这是因为数据增强让训练集变得"更难"，模型被迫学习不变性，反而在干净的测试集上表现更好。这是泛化能力的标志。
